The advantage of using an OpenStack server to host your VM on is that you can isolate your VM from the inconsistencies between your own local machine and anyone else's.
Eventually, this web application may be able to deploy the VM on OpenStack from this GUI interface itself, instead of relying on an executable output source code. This way, the VMs could also be timed to be short-lived, to reduce the number of idle VMs taking up server resources.

Access your VM
$ ssh root@YOUR_VM_IP

To access your VM, you simply need to call ssh with the IP of your instance, which can be found on DevStack.

Installation System Requirements: In order for the scripts to run successfully, VirtualBox, Vagrant and Git version control (for cloning Stash repositories) must be installed on your computer.


Manage your VM on OpenStack
  1. Log in to DevStack with the username as 'admin' and the password being 'orionsys'.
  2. Go to 'project', and click 'instances'.
  3. You may manually terminate your instance from this GUI panel, and it is recommended that you do so if you have finished with your VM to save on server resources.

Configuring Puppet Modules

To configure the installation of OHP using Puppet modules, the parameters may be set in the manifest file manifests/site.pp. The documentation on the options for the parameters are found in the manifest file modules/puppet-ohp/ohp/manifests/init.pp. More details tips on configuration of Puppet manifest files can be found on this Woki page.

FAQ#1

The entire booting and provisioning process may take around 30 minutes depending on the speed of your host machine and internet connection. This is mostly due to the creation of the solution.zip, which takes about 20 minutes.
If you're impatient, check out the Vagrantfile and up the allocated memory.
