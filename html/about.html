About Give-Me-A-Deploy

Give-Me-A-Deploy web-based GUI application that generates scripts which automatically create, configure and provision Virtual Machines (a.k.a. VMs). It uses the Vagrant command line VM manager with its built-in Puppet automation tool to install and configure Orion Health products using the same OHP Puppet modules used in production. It aims to make VMs as quick and easy as possible for users at Orion Health, as they provide ideal, consistent environments for development and testing.

For more information, please have a read of these presentation slides and the lightning talk from the intern projects demo.

Next up: Short-lived VMs, built on-demand

While the current Give-Me-A-Deploy is ideal for spinning up VMs locally using custom generated source code, it would be even better to have provisioned VMs spun up in a public server such as HP Cloud, straight from our GUI interface. A third party server would allow developers and testers to focus on their applications and code without worrying about potential server issues.

By making these on-demand VMs also short-lived (automatically expiring after 48 hours), we would also eliminate the problem of idle VMs wasting server resources while continuing to provide an efficient solution to the demand for fast and flexible development environments.
It's less time setting up your environments, and more time working in them.
